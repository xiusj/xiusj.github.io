kubernetes client-go介绍

2017-09-24

-次访问

client-go

client-go是k8s官方的client库，主要用于连接k8s的apiserver，封装对k8s的所有的resource的操作，如果想对在k8s现有的基础上做二次开发，各人觉得非常有必要对其有一定的了解。有的人说，k8s不是有restful的api接口吗？我直接调用它就就可以了。没错，是有完善的restful的api接口，但是其灵活性还是远远不够的。

目的

熟悉client-go的代码的目的有两个：

  1. 通过代码的方式实现对k8s的各种资源的操作，在此基础上对k8s的各种资源的熟悉以及强化记忆
  2. 通过对代码的仔细阅读，理解k8s对各种资源的存储。以及k8s是怎样实现各种资源的内存存储的
  3. 通过对代码的熟悉，帮助很好的list-wath机制

项目整体目录

  • discovery 还没细看
  • dynamic 还没细看
  • example client-go的使用例子
  • informers 对内部存储的再次封装
  • kubernetes k8s对应的所有的资源的所有的操作方法的接口封装，包括增删改查 lister,wather等
  • listers k8s对应的所有的资源的lister 的接口封装
  • pkg client go自己用到的工具类
  • plugin 第三方插件的client端封装，包括认证 openstack,gcp等
  • rest 实现http请求的链式操作，发送请求到对应的k8s的restful api
  • testing 所有测试文件
  • third-party 封装golang的一些库
  • tools 认证，内部存储，client端的请求封装，service的端口转发的定义，分页的封装
  • transport tls安全类型的https实现的一些封装，包括x590证书的生成等
  • util 封装的一些工具类
